Running from Source

To compile and run from source code:

# Clone the repository
git clone https://github.com/Fatherly-P-Titus/music-player-app.git

# Navigate to project directory
cd music-player-app

# Compile the project
javac --module-path /path/to/javafx-sdk/lib \
      --add-modules javafx.controls,javafx.fxml,javafx.media \
      -d out src/**/*.java

# Run the application
java --module-path /path/to/javafx-sdk/lib \
     --add-modules javafx.controls,javafx.fxml,javafx.media \
     -cp out com.musicplayer.Main

Frequently Asked Questions

Why can't I see my music files?

Ensure your audio files are in a supported format (MP3, WAV, M4A) and located in a directory that the app has permission to access. Use the folder scanning feature to add your music directories.

The app crashes when playing certain files

This might be caused by corrupted audio files or unsupported codecs. Try playing the file in another media player to verify it's not corrupted.

No sound from the application

Check your system volume, ensure the app volume slider is up, and verify that no other application is exclusively using your audio device.

Performance issues with large libraries

The app is optimized for performance, but very large libraries (10,000+ songs) may experience slower loading times. Consider organizing your music into separate folders.

Troubleshooting Common Issues

JavaFX Not Found

If you encounter errors related to JavaFX modules, ensure you've properly set the module path:

--module-path /path/to/javafx-sdk/lib \
--add-modules javafx.controls,javafx.fxml,javafx.media

Audio Playback Issues

If audio doesn't play, check that:

  1. Your system audio is working
  2. No other app is blocking audio output
  3. File permissions allow reading the audio files
  4. Audio files are not corrupted
